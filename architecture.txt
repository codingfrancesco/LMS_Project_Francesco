1. PROJECT OVERVIEW
================================================================================
The LMS is a comprehensive web-based Learning Management System built with Flask,
enabling educators to create courses, manage lessons, and track student progress,
while allowing students to enroll in courses, complete assignments, and view grades.

Technology Stack:
  - Backend Framework: Flask (Python web framework)
  - Database: SQLite (local relational database)
  - Frontend: HTML5, CSS, Jinja2 templating
  - Server: Flask development/production server
  - Session Management: Flask sessions with secret key encryption

3. DATABASE SCHEMA
================================================================================

TABLE: users
  - id (INTEGER, PRIMARY KEY, AUTO-INCREMENT)
  - username (TEXT, NOT NULL, UNIQUE)
  - email (TEXT, NOT NULL, UNIQUE)
  - password (TEXT, NOT NULL, SHA256-hashed)
  - full_name (TEXT, NOT NULL)
  - role (TEXT, DEFAULT 'student') - Values: 'student', 'teacher', 'admin'
  - created_at (TIMESTAMP, AUTO)
  - last_login (TIMESTAMP)
  
  Purpose: Store user account information

TABLE: courses
  - id (INTEGER, PRIMARY KEY, AUTO-INCREMENT)
  - course_id (INTEGER)
  - title (TEXT, NOT NULL, UNIQUE)
  - description (TEXT)
  - course_type (TEXT, DEFAULT 'Self-Paced') 
    Values: 'Self-Paced', 'Instructor-Led', 'Hybrid', 'Workshop'
  - duration (TEXT, DEFAULT 'Flexible')
  - level (TEXT, DEFAULT 'Beginner')
    Values: 'Beginner', 'Intermediate', 'Advanced', 'Expert'
  - teacher_id (INTEGER, FOREIGN KEY -> users(id))
  - created_at (TIMESTAMP, AUTO)
  
  Purpose: Store course information and metadata

TABLE: topics (lessons)
  - id (INTEGER, PRIMARY KEY, AUTO-INCREMENT)
  - course_id (INTEGER, NOT NULL, FOREIGN KEY -> courses(id))
  - title (TEXT, UNIQUE)
  - subtitle (TEXT)
  - content (TEXT) - Main lesson body/content
  - created_at (TIMESTAMP, AUTO)
  
  Purpose: Store lesson topics/chapters within courses

TABLE: msqs (Multiple-choice Questions)
  - id (INTEGER, PRIMARY KEY, AUTO-INCREMENT)
  - topic_id (INTEGER, NOT NULL, FOREIGN KEY -> topics(id))
  - question (TEXT, NOT NULL)
  - option_a (TEXT, NOT NULL)
  - option_b (TEXT, NOT NULL)
  - option_c (TEXT, NOT NULL)
  - option_d (TEXT, NOT NULL)
  - correct_answer (TEXT, NOT NULL) - Values: 'a', 'b', 'c', 'd'
  - created_at (TIMESTAMP, AUTO)
  
  Purpose: Store multiple-choice questions for assignments

TABLE: enrollments
  - id (INTEGER, PRIMARY KEY, AUTO-INCREMENT)
  - student_id (INTEGER, NOT NULL, FOREIGN KEY -> users(id))
  - course_id (INTEGER, NOT NULL, FOREIGN KEY -> courses(id))
  - progress (INTEGER, DEFAULT 0) - Progress percentage (0-100)
  - enrolled_at (TIMESTAMP, AUTO)
  - UNIQUE(student_id, course_id) - Each student enrolls once per course
  
  Purpose: Track student enrollment in courses

TABLE: submissions
  - id (INTEGER, PRIMARY KEY, AUTO-INCREMENT)
  - student_id (INTEGER, NOT NULL, FOREIGN KEY -> users(id))
  - question_id (INTEGER, NOT NULL, FOREIGN KEY -> msqs(id))
  - selected_answer (TEXT, NOT NULL) - Values: 'a', 'b', 'c', 'd'
  - is_correct (INTEGER, DEFAULT 0) - 1 for correct, 0 for incorrect
  - submitted_at (TIMESTAMP, AUTO)
  
  Purpose: Store student quiz/assignment submissions and answers

TABLE: attendance
  - id (INTEGER, PRIMARY KEY, AUTO-INCREMENT)
  - student_id (INTEGER, NOT NULL, FOREIGN KEY -> users(id))
  - lesson_id (INTEGER, NOT NULL, FOREIGN KEY -> topics(id))
  - course_id (INTEGER, NOT NULL, FOREIGN KEY -> courses(id))
  - status (TEXT, DEFAULT 'absent') - Values: 'present', 'absent', 'late'
  - lesson_date (TIMESTAMP, AUTO)
  - recorded_at (TIMESTAMP, AUTO)
  
  Purpose: Track student attendance for lessons

TABLE: notifications
  - id (INTEGER, PRIMARY KEY, AUTO-INCREMENT)
  - student_id (INTEGER, NOT NULL, FOREIGN KEY -> users(id))
  - course_id (INTEGER, NOT NULL, FOREIGN KEY -> courses(id))
  - notification_type (TEXT, NOT NULL) - Values: 'lesson', 'assignment'
  - title (TEXT, NOT NULL)
  - message (TEXT)
  - resource_id (INTEGER) - Link to lesson_id or assignment_id
  - is_read (INTEGER, DEFAULT 0)
  - created_at (TIMESTAMP, AUTO)
  
  Purpose: Notify students of new lessons, assignments, and updates

TABLE: grades
  - id (INTEGER, PRIMARY KEY, AUTO-INCREMENT)
  - student_id (INTEGER, NOT NULL, FOREIGN KEY -> users(id))
  - assignment_id (INTEGER, NOT NULL, FOREIGN KEY -> assignments(id))
  - teacher_id (INTEGER, NOT NULL, FOREIGN KEY -> users(id))
  - grade (REAL) - Numeric grade (0-100)
  - feedback (TEXT) - Teacher's comments/feedback
  - graded_at (TIMESTAMP, AUTO)
  - updated_at (TIMESTAMP)
  
  Purpose: Store assignment grades and teacher feedback

TABLE: comments
  - id (INTEGER, PRIMARY KEY, AUTO-INCREMENT)
  - user_id (INTEGER, NOT NULL, FOREIGN KEY -> users(id))
  - course_id (INTEGER, NOT NULL, FOREIGN KEY -> courses(id))
  - message (TEXT, NOT NULL)
  - created_at (TIMESTAMP, AUTO)
  
  Purpose: Course discussions and student-teacher interactions


4. CORE COMPONENTS & FUNCTIONALITY
================================================================================

4.1 AUTHENTICATION & AUTHORIZATION
-----------------------------------
Files: app.py (main authentication logic)

Components:
  - Session Management: Flask sessions with SECRET_KEY encryption
  - Password Security: SHA256 hashing for password storage
  - User Roles: student, teacher, admin
  - Route Protection: @login_required decorators on protected routes

Key Functions:
  - hash_password(password): Hash user passwords securely
  - get_db_connection(db_name): Establish database connection with Row factory
  - init_db(): Initialize all database tables on first run

4.2 COURSE MANAGEMENT
-----------------------------------
Files: app.py, scripts/course_table.py

Teacher Capabilities:
  - Create new courses (with type, duration, level, description)
  - Edit course details
  - Manage course content and structure
  - View enrolled students
  - Track course progress

Student Capabilities:
  - Browse available courses
  - Enroll in courses
  - Track personal progress
  - View enrolled courses

Database: courses, enrollments tables

4.3 CONTENT MANAGEMENT (LESSONS/TOPICS)
-----------------------------------
Files: app.py, scripts/topic_table.py

Teacher Capabilities:
  - Create lessons within courses
  - Edit lesson content and topics
  - Create multiple-choice quiz questions
  - Delete lessons
  - Structure course material

Student Capabilities:
  - View lesson content
  - Read lesson materials
  - Take quiz/assignments

Database: topics (lessons), msqs (questions) tables

4.4 ASSIGNMENT & GRADING SYSTEM
-----------------------------------
Files: app.py, scripts/assignments_table.py, scripts/msq_table.py

Teacher Capabilities:
  - Create assignment questions (MCQ format)
  - View student submissions
  - Grade submissions with feedback
  - Provide feedback comments
  - View assignment statistics

Student Capabilities:
  - View available assignments
  - Submit answers to questions
  - View results and grades
  - See teacher feedback

Database: msqs, submissions, grades tables

4.5 ATTENDANCE TRACKING
-----------------------------------
Files: app.py

Teacher Capabilities:
  - Mark student attendance for lessons
  - Record attendance status (present, absent, late)
  - View attendance reports
  - Generate attendance statistics

Student Capabilities:
  - View own attendance record

Database: attendance table

4.6 NOTIFICATION SYSTEM
-----------------------------------
Files: app.py

Features:
  - Notify students when new lessons are created
  - Alert students about new assignments
  - Mark notifications as read/unread
  - Display unread notification count

Database: notifications table

4.7 DISCUSSION/COMMENTS SYSTEM
-----------------------------------
Files: app.py

Features:
  - Students can post comments on courses
  - Teachers can respond to student comments
  - Build course community and discussion
  - Thread-like interaction

Database: comments table


5. API ROUTES & ENDPOINTS
================================================================================

5.1 AUTHENTICATION ROUTES
-----------------------------------
GET/POST /register        - User registration (student/teacher)
GET/POST /login           - User login
GET      /logout          - User logout

5.2 DASHBOARD ROUTES
-----------------------------------
GET      /student_dashboard    - Student main dashboard view
GET      /teacher_dashboard    - Teacher main dashboard view
GET      /admin_dashboard      - Admin management dashboard
GET      /                     - Home page with statistics

5.3 COURSE MANAGEMENT ROUTES
-----------------------------------
GET/POST /create_course         - Create new course
GET/POST /manage_course/<id>    - Manage existing course
GET      /course                - View course list/details
GET      /learn/<course_id>     - Enroll and view course content
GET      /enroll/<course_id>    - Enroll student in course

5.4 LESSON/CONTENT ROUTES
-----------------------------------
GET/POST /create_lesson/<course_id>     - Create lesson
GET/POST /edit_lesson/<lesson_id>       - Edit lesson
POST     /delete_lesson/<lesson_id>     - Delete lesson
GET      /lessons                       - View all lessons
GET      /lesson/<lesson_id>            - View lesson detail page

5.5 ASSIGNMENT ROUTES
-----------------------------------
GET/POST /create_assignment/<course_id> - Create assignment questions
GET      /assignments                   - View assignments list
POST     /delete_assignment/<id>        - Delete assignment
GET      /grade_assignment/<id>         - View submissions for grading
POST     /grade_submission/<id>         - Submit grade and feedback

5.6 STUDENT ACTIVITY ROUTES
-----------------------------------
GET      /my_assignments         - View student's assignments
GET      /student_grades         - View grades and feedback
GET      /assignment_results     - View submission results
GET      /notifications          - View student notifications

5.7 ATTENDANCE ROUTES
-----------------------------------
GET/POST /mark_attendance/<lesson_id>  - Mark attendance
GET      /attendance_report             - View attendance report


6. DATA FLOW & USER INTERACTIONS
================================================================================

6.1 STUDENT ENROLLMENT & LEARNING FLOW
-----------------------------------
1. Student registers account (role='student')
2. Student logs in with credentials
3. Student browses available courses
4. Student enrolls in course (creates enrollment record)
5. Student accesses course (learn_course view)
6. Student views lessons and reads content
7. Student completes quiz questions (submissions)
8. System records answers and evaluates correctness
9. Teacher grades submissions and provides feedback
10. Student views grades and feedback in dashboard
11. Notifications alert student of new content

6.2 TEACHER COURSE CREATION FLOW
-----------------------------------
1. Teacher registers account (role='teacher')
2. Teacher logs in
3. Teacher creates new course (with metadata)
4. Teacher creates lessons/topics within course
5. Teacher creates quiz questions (MCQ format)
6. Teacher publishes course content
7. Students can now enroll in course
8. Teacher views enrolled students
9. Teacher reviews student submissions
10. Teacher grades submissions with feedback
11. Teacher marks attendance
12. Teacher tracks course statistics

6.3 ADMINISTRATIVE FLOW
-----------------------------------
1. Admin accesses admin_dashboard
2. Admin manages system statistics
3. Admin can create user accounts
4. Admin monitors system health
5. Admin generates reports


7. SECURITY FEATURES
================================================================================

- Password Hashing: SHA256 algorithm for secure password storage
- Session Management: Flask session encryption with SECRET_KEY
- Input Validation: Request validation in forms
- Database Security: Parameterized queries to prevent SQL injection
- Role-Based Access Control: Routes check user role (student/teacher/admin)
- HTTPS Ready: Structure supports HTTPS deployment

Note: Production deployment should use:
  - Environment variables for SECRET_KEY
  - HTTPS/SSL certificates
  - Production database (PostgreSQL, MySQL)
  - Web server (Gunicorn, uWSGI)


8. KEY TECHNOLOGIES & LIBRARIES
================================================================================

Backend:
  - Flask 2.3.3: Web framework for HTTP routing and request handling
  - Werkzeug 2.3.7: WSGI utilities for Flask
  
Built-in Libraries:
  - sqlite3: Database operations and schema management
  - hashlib: Password hashing with SHA256
  - os: Environment variable management
  
Frontend:
  - HTML5: Markup and semantic structure
  - CSS3: Styling and responsive design
  - Jinja2: Template engine (Flask default)
